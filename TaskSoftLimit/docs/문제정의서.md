# 문제 정의서 — 목표 관리 서비스

## 1. 관련 지표

| 지표 | 설명 |
|------|------|
| **Day-3 유지율** | 가입 후 3일째에도 To Do를 생성하거나 체크하는 유저 비율 |
| **과업 성공률** | (완료한 일 수 / 생성한 전체 할 일 수)의 평균치 |
| **번아웃 임계치 비율** | 미완료 항목이 5개 이상 쌓인 유저의 비율 (이 수치를 낮추는 것이 핵심) |
| **주간 성취 빈도** | 일주일 중 목표를 100% 달성한 날의 수 (성취 경험에 집중) |

## 2. 목표

- 사용자 고민(해석) 없이 즉시 수행 가능한 **숫자/시간 기반의 행동 단위 목표**를 작성하도록 가이드
- 절대 수치가 아닌 **'자기의 나' 대비 변화된 데이터**를 통해 성장을 시각적으로 인지하게 함
- 미완료 상향을 실패로 판단하지 않고, **보관/휴식/전환** 등 관리할 수 있도록 함

## 3. 문제 배경

- To-Do List 작성 시 **초기 목표 설정 이후 3일 이내** 작성을 중단하는 비율이 높고, 목표 완료율도 저조함
- 특히 **미완료 항목이 5개 이상** 누적될 경우 리스트 열람 빈도 급감 및 재방문 간격이 길어지는 등 서비스 이용 활동이 유의미하게 감소하는 패턴이 확인됨
- 이는 다음으로 해석됨:
  - 개인 역량을 고려하지 않은 **모호한 목표 설정**
  - 오늘의 일이 장기적 성취와 **연결되지 않는 구조 인지 실패**로 인한 목표 수행 동력 상실
  - 미완료 과업 이월 시 발생하는 **심리적 부담**이 부정적 정서로 작용하여 불쾌한 경험 회피로 이어짐

## 4. 문제 정의

- 사용자는 **목표 달성을 통한 성장 경험을 체감하지 못하고**, 서비스는 '성장 도구'가 아닌 **'부담 요소'**로 인식됨
- 결과적으로 **일일 목표 관리 기능의 지속 사용성이 저하**되고, 서비스 재방문 포기 및 이탈로 이어짐
- To-Do List 사용자는 초기 목표 설정 이후 **지속적인 동기를 유지하지 못하고**, 미완료 항목 누적에 따른 **심리적 부담**으로 서비스 이용을 중단하는 경향이 있음
- 이는 **사용자의 역량에 맞는 실현 가능한 목표 설정**과, **목표 수행 결과를 긍정적으로 해석할 수 있는 기준 부재** 때문이며, 목표 수행 지속성을 저해하는 문제로 이어짐
- 목표 실패 이후, **사용자가 자신의 상태를 제대로 파악하지 못함**

## 5. 사용자 관측 (Pain Point)

### 정량적 관측

- 미완료 항목 누적으로 인한 **Day-3 Retention 40% 미만** (초기 진입자 60% 이탈)
- **미완료 과업 5개 초과 시** 앱 접속 빈도 이전 대비 **50% 이하로 급감**
- 일일 계획 달성률 평균 **50% 미만**
- 부정적 감정(죄책감)을 회피하기 위해 **새로운 목표 생성 지체**를 유발

### 정성적 관측

- 주 4회 이상 계획을 세우는 '고관여 유지' 조치 달성률이 낮을 경우 **이탈 속도가 더 가속화**됨을 확인

### 세부 행동 및 문제점

| 구분 | 내용 | 관측 지표 |
|------|------|------------|
| **압박과 회피** | 미완료 항목에서 심리적 부담을 인지하고, 리스트 열람 직후 앱 종료, 접속 미루기 등 회피 행동 → 재방문 간격 증가 → 서비스 방문 중단 | 리스트 진입 후 즉시 종료, 평균 재방문 간격, 미완료 항목 수 기준 코호트 리텐션(7일/14일) |
| **목표 난이도 구체성 실패** | 목표가 행동 단위로 구체화되지 않거나 난이도 부적절 → 기계적 완료 또는 반복적 실패 → 목표 수행 경험의 의미 약화 → 사용 감소 | 목표 달성률 구간별 이탈률, 수치/시간 키워드 포함률, 동일 문구 목표 제시 사용률 |
| **성장 인식 부재** | 완료 기록 사이에서 자신의 성장 누적을 인식하지 못함, 성취 관련 정보(통계 최고) 미확인 → 서비스 필요성 미체감 → 재방문 중단 | 통계 최고 화면 조회율, 목표 완료 후 24시간 내 재방문, 완료 개수 대비 누적 사용 기간 |

## 6. 제약 조건 (Solution Constraints)

- **심리적 가드레일:** 미완료 항목이 일정 수 이상 누적되면, 새로운 할 일 추가를 **제한**하거나 목표 하향 조정을 **'강제'가 아닌 '권유'** 형태로 제안 (예: 3, 5, 7개)  
  → *ActiveTaskCount 기반 Soft 제한 실험: 생성은 허용하되 가이드만 노출*
- **입력 최소화:** 학습적인 To-Do 작성 행위가 부담이 되지 않도록, **입력 시간 2분 이내** 제한
- **정답 지향 금지:** AI가 유저를 '평가'하거나 '혼내는' 뉘앙스를 주어서는 안 되며, **객관적인 상태만** 보여줌
- **부채형 UI 금지:** 미완료 항목이 현재 행동을 방해하지 않도록, **과거 기록과 오늘의 행동 영역을 명확히 분리**  
  → *UX 요소 구현 금지 (팝업, UI 코드 금지)*
- **성공 기준:** 미완료 항목 처리 없이 오늘의 목표를 달성하면 **항상 '쉬운 접근' 기능** 제공
- **강제적 의사결정 요구 금지:** 미완료 항목에 대한 처리 결정을 **서비스 이용의 전제 조건으로 요구하지 않음**  
  → *생성은 허용하되 가이드만 노출 (Soft 제한)*
